# Knot Project Structure Analysis

Generated by bk-analyze on 2026-02-19

## Overview

Knot is a Tauri 2.0 application with a React/TypeScript frontend and Rust backend. It provides Obsidian-like note-taking with a distraction-free editor, full-text search, and link graph visualization.

## Data Flow

### Opening a Vault

1. User enters path in UI
2. `openVault(path)` API call
3. `commands/vault.rs::open_vault` handler
4. `VaultManager::open()` - loads config, DB, search index
5. `rebuild_graph()` - loads links from DB
6. Returns `VaultInfo` to frontend
7. Zustand store updated
8. `loadNotes()` called
9. Note list populated in sidebar

### Saving a Note

1. User presses Ctrl+S
2. `handleSave()` in Editor component
3. `api.saveNote(path, content)`
4. `commands/notes.rs::save_note` handler
5. `VaultManager::save_note()`:
   - Write to filesystem
   - Update database
   - Update search index (`index_note`)
   - Update graph (`update_graph_for_note`)
6. Returns success
7. Editor dirty flag cleared
8. Note list refreshed

### Searching

1. User types query
2. `api.searchNotes(query)`
3. `commands/search.rs::search_notes` handler
4. `VaultManager::search()` → `SearchIndex::search()`
5. Query parsed for advanced syntax
6. Tantivy search executed
7. Snippets generated
8. Results returned to frontend

## Security Boundaries

1. **File system access**: Restricted to vault directory and `.vault/` subdirectory
2. **No network access**: Local-only application (P2P sync planned for future)
3. **Plugins**: Conditionally compiled, disabled by default (`plugins_enabled: false`)

## Performance Considerations

1. **Search index**: In-memory Tantivy with 50MB writer buffer
2. **Graph**: In-memory petgraph, rebuilt on vault open and note changes
3. **Database**: SQLite with WAL mode for better concurrency
4. **Markdown parsing**: Two-pass (wikilinks preprocessing, then pulldown-cmark)
